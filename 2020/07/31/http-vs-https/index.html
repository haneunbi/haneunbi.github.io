[Web] HTTP와 HTTPS의 차이

HTTP

HTTP(HyperText Transfer Protocol)는 주로 HTML(HyperText Markup Language) 문서를 주고받는 데에 쓰이며, 클라이언트(웹 브라우저)와 서버 사이에 이루어지는 요청/응답(request/response) 프로토콜이다.

HTTPS

HTTPS(Hypertext Transfer Protocol Secure)는 HTTP의 보안 취약점을 해결하기 위한 프로토콜로 데이터를 암호화된 방식으로 주고받는 통신 규약을 말한다. SSL(Secure Sockets Layer, 보안 소켓 계층)과 TLS(Transport Layer Security, 전송 계층 보안)을 사용하여 보안 취약점을 해결한다. 또한 검색엔진 최적화(SEO)에 있어서도 구글은 HTTPS 웹 사이트에 가산점을 준다.

프로토콜(Protocol)

통신 시스템이 데이터를 교환하기 위해 사용하는 통신 규약이다.

소켓(Socket)

소켓은 네트워크를 경유하는 프로세스 간 통신의 종착점으로 소켓 통신은 HTTP 통신과 달리 네트워크 상에서 서버와 클라이언트가 특정 포트를 통해 양방향 통신을 하는 방식(소프트웨어 장치)이다.

주요 HTTP 응답 상태 코드

클라이언트가 서버에 어떠한 요청(request)을 하면, 서버는 세 자리 수로 된 상태 코드와 함께 응답(response)한다.

코드 설명
200 요청 성공
302 페이지 이동
401 인증 실패
403 접근 금지
404 페이지 없음
500 서버 에러

여기서 말하는 2xx번대, 4xx번대 숫자 그룹이 의미하는 바는 아래와 같다.

코드 의미 설명
1 Informational(정보) 요청을 받고 처리 중에 있음
2 Success(성공) 요청을 정상적으로 처리함
3 Redirection(리다이렉션) 요청 완료를 위해 추가 동작이 필요함
4 Client Error(클라이언트 에러) 클라이언트의 요청을 처리할 수 없어 에러 발생
5 Server Error(서버 에러) 서버가 요청을 처리할 수 없어 에러 발생

왜 알아야 할까?

HTTPS 페이지에서 HTTP 프로토콜을 사용하여 리소스(이미지, 동영상, 스타일시트 등)를 요청할 경우 혼합 콘텐츠가 발생하는데 콘솔에 아래와 같이 ‘보안되지 않은 리소스가 포함되어 있음'을 사용자에게 경고해준다.

Mixed Content: The page at 'https://URL' was loaded over HTTPS, 
but requested an insecure resource 'http://resource-URL'. 
This request has been blocked; the content must be served over HTTPS.

위 오류에 대한 가장 쉬운 해결 방법은 리소스 경로 작성 시 http: 혹은 https: 를 제외하고 작성하면, 현재 웹 페이지의 프로토콜이 자동으로 붙어서 요청된다. 단, HTTPS를 통해 사용할 수 없는 리소스의 경우는 다른 호스트의 리소스로 변경하거나 리소스를 프로젝트 내에 직접 다운로드 후 호스팅한다.

<img src="http://URL">

<!-- http: 혹은 https: 를 제외하고 작성 -->
<img src="//URL">

@import url(//fonts.googleapis.com/earlyaccess/kopubbatang.css);

▲ 참고로 Google Fonts Early Access에서 폰트 경로를 봐도 프로토콜을 제외한 후 작성되어 있는 것을 확인해볼 수 있다.

[Web] 검색 엔진 최적화(SEO)란 무엇인가?

검색 엔진 최적화(Search Engine Optimization, SEO)란 사이트 내 콘텐츠 정보를 검색 엔진이 잘 이해할 수 있도록 정리하는 것으로 검색 결과의 상위에 나올 수 있도록 하는 작업을 말한다. 각 포털사는 검색 엔진에 친화적인 사이트를 우대하기 때문에 사이트의 방문자 유입 증가를 위한 필수적인 작업이다.

사이트가 포털에 검색되는 방식은,
검색 엔진의 크롤러(검색 로봇)가 웹을 크롤링(신규/업데이트된 웹페이지를 찾는 프로세스)하면서 페이지에서 페이지로 링크를 따라가고, 찾은 콘텐츠의 색인(해당 페이지의 콘텐츠 및 URL 명시)을 생성한다. 검색 결과에 보이는 것은 바로 그 콘텐츠 색인이다.

콘텐츠 마크업

검색 엔진에 친화적인 사이트를 구축하기 위해서는 웹표준에 적합한 HTML 마크업을 사용해야 한다. 또한 검색 엔진이 참고하는 정보는 사용자의 눈에 보이는 콘텐츠도 포함되지만 사용자의 눈에 보이지 않는 메타 영역도 존재한다.

선호 URL (=대표 URL) 지정

URL 표현 시에는 상대 경로가 아닌 절대 경로를 사용해야 한다.

<head>
<link rel="canonical" href="http://www.mysite.com/article/article1.html">
</head>

로봇 메타 태그

로봇 메타 태그는 페이지 별로 검색 로봇의 접근 여부를 제어할 수 있는 정보이다.

<head>
<!-- 기본 설정 (태그를 쓰지 않아도 무방) -->
<meta name="robots" content="index,follow"> 

<!-- 색인 대상으로 처리, 페이지 내 링크 미 수집 -->
<meta name="robots" content="index,nofollow">

<!-- 색인 대상에서 제외, 페이지 내 링크 수집 -->
<meta name="robots" content="noindex,follow">

<!-- 색인 대상에서 제외, 페이지 내 링크 미 수집 -->
<meta name="robots" content="noindex,nofollow">
</head>

HTML 문서의 head 태그 내에 있는 meta 태그 외에 검색 로봇에게 사이트 및 웹페이지를 수집할 수 있도록 허용하거나 제한하는 국제 권고안인 robots.txt 파일을 두는 방법이 있으며, 만약 사이트의 루트 디렉터리에 robots.txt 파일이 없다면 모든 콘텐츠를 수집할 수 있도록 간주한다.
robot.txt 설정 자세히 보기

페이지 제목

페이지 콘텐츠에 맞는 고유한 제목을 기입한다.

<head>
<title>페이지 제목</title>
</head>

페이지 설명

페이지의 콘텐츠에 대한 간략한 설명으로 1-2개의 문장으로 구성된 짧은 단락을 사용한다.

<head>
<meta name="description" content="페이지 설명">
</head>

오픈 그래프 (Open Graph)

오픈 그래프 태그는 사이트가 소셜 미디어로 공유될 때 우선적으로 활용되는 정보이다. 사이트의 제목, 설명과 더불어서 사이트를 대표하는 이미지(권장 해상도: 1200x627 px, 1.91:1))도 같이 넣어준다. (자세한 내용은 http://ogp.me 참고)

<head>
<meta property="og:type" content="website"> 
<meta property="og:title" content="페이지 제목">
<meta property="og:description" content="페이지 설명">
<meta property="og:image" content="http://www.mysite.com/myimage.jpg">
<meta property="og:url" content="http://www.mysite.com">
</head>

트위터 카드 (Twitter Cards)

<head>
<meta name="twitter:card" content="summary">
<meta name="twitter:title" content="페이지 제목">
<meta name="twitter:description" content="페이지 설명">
<meta name="twitter:image" content="http://www.mysite.com/myimage.jpg">
<meta name="twitter:url" content="http://www.mysite.com">
</head>

  • 트위터 카드 검증 | Twitter Developer

파비콘 (Favorites icon, Favicon)

파비콘이 위치(루트 디렉토리)하는 href 에는 상대 경로가 아닌 절대 경로를 사용해야 하며 파비콘 사이즈는 너비와 높이가 동일한 정사각형이어야 한다. (16x16, 32x32 px) 파일 포맷 참고

<head>
<!-- IE10 이하 -->
<link rel="shortcut icon" href="http://www.mysite.com/favicon.ico">
 
<!-- 브라우저용 표준 (IE11+)-->
<link rel="icon" type="image/png" sizes="32x32" href="http://www.mysite.com/favicon-32.png">
 
<!-- Android -->
<link rel="icon" href="http://www.mysite.com/favicon-192.png">
 
<!-- Apple -->
<link rel="apple-touch-icon" href="http://www.mysite.com/favicon-57.png">
<link rel="apple-touch-icon" sizes="152x152" href="http://www.mysite.com/favicon-152.png"> <!-- iPad -->
<link rel="apple-touch-icon" sizes="167x167" href="http://www.mysite.com/favicon-167.png"> <!-- iPad Retina -->
<link rel="apple-touch-icon" sizes="180x180" href="http://www.mysite.com/favicon-180.png"> <!-- iPhone 6 plus+ (IOS8+) -->
</head>

모바일 사용성

반응형 웹

반응형 웹은 웹 브라우저가 웹 문서의 가로폭을 기기의 스크린 크기에 맞게 자동적으로 조절하는 기법이다.

<head>
<meta name="viewport" content="width=device-width">
</head>

별도의 모바일 URL 제공

동일한 콘텐츠에 대하여 데스크톱 사이트에 대응하는 별도의 모바일 사이트가 있다면 모바일 사이트의 페이지에 1:1로 대응되는 데스크톱 사이트의 URL을 명시적으로 지정하는 것을 권장한다.

데스크톱 사이트로 http://www.mysite.com이 있고 그와 대응하는 모바일 사이트인 http://m.mysite.com이 있다고 가정한다면 http://m.mysite.com 내의 페이지에 아래와 같이 데스크톱의 URL을 명시하는 것이 좋다.

<head>
<link rel="canonical" href="http://www.mysite.com/">
</head>

▲ 모바일 사이트의 메인 페이지(http://m.mysite.com/)에 데스크톱 사이트의 URL 지정

<head>
<link rel="canonical" href="http://www.mysite.com/article/article1.html">
</head>

▲ 모바일 사이트의 개별 웹 페이지(http://m.mysite.com/article/article1.html)에 데스크톱 사이트의 URL 지정

모바일 중심 색인 생성 ⭐

구글은 색인을 생성하고 순위를 지정하는 데 모바일 버전의 콘텐츠를 우선적으로 사용한다. 또한 구글은 페이지 로딩 속도를 검색 순위에 사용하기 때문에 구글이 제공하는 모바일 친화성 테스트와 속도 테스크를 통해 점검해 보자.

구글이 좋아하는 웹 사이트의 3대 조건

  1. 보안 프로토콜이 적용된 웹 사이트
  2. 모바일 친화성을 갖춘 웹 사이트
  3. 페이지 로딩 속도가 빠른 웹 사이트

SEO 마크업 예시

<html lang="ko">
<head>
    <meta http-equiv="X-UA-Compatible" content="IE=edge">
    <meta charset="utf-8">
    <title>페이지 제목</title>
    <meta name="viewport" content="width=device-width,initial-scale=1">
    <meta name="description" content="페이지 설명">

    <!-- Open Graphy -->
    <meta property="og:type" content="website">
    <meta property="og:title" content="페이지 제목">
    <meta property="og:description" content="페이지 설명">
    <meta property="og:image" content="http://www.mysite.com/myimage.jpg">
    <meta property="og:url" content="http://www.mysite.com">

    <!-- Twitter Card -->
    <meta name="twitter:card" content="summary">
    <meta name="twitter:title" content="페이지 제목">
    <meta name="twitter:description" content="페이지 설명">
    <meta name="twitter:image" content="http://www.mysite.com/myimage.jpg">
    <meta name="twitter:url" content="http://www.mysite.com">

    <link rel="shortcut icon" href="http://www.mysite.com/favicon.ico">
    <link rel="canonical" href="http://www.mysite.com">
</head>

<body>
    <div class="wrap">
        <!-- 콘텐츠 작성 -->
    </div> <!-- //wrap -->
</body>

</html>

가장 처음에 나오는 호환성 보기 관련 메타 태그에 대해 부연 설명을 하자면,

<head>
    <meta http-equiv="X-UA-Compatible" content="IE=edge"> <!-- 1. 호환성 보기 -->
    <meta charset="utf-8"> <!-- 2 -->
    <title>페이지 제목</title> <!-- 3 -->
    ...
</head>
  1. IE 브라우저에서 최신 표준 모드로 렌더링하도록 설정한다.
    (호환성 보기 메타 태그는 head 태그의 가장 처음에 위치해야 함)
  2. 모든 브라우저에서 사용할 인코딩을 설정한다.
  3. 위 정의 된 인코딩으로 페이지 제목을 출력한다.

마지막으로 한가지 드는 의문점 ❓

기본 title 태그와 description 메타 태그가 있을 경우, 오픈 그래프와 트위터 카드의 메타 태그가 각각 꼭 필요할까?

Facebook이 페이지에서 og 태그를 찾지 못하면 대신 메타의 제목과 설명을 사용하고, Twitter 역시 Twitter Card 관련 태그를 먼저 확인하고 존재하지 않는 경우 지원되는 오픈 그래프 속성으로 대체한다고 하는데..

그러면 중복 기능을 하는 태그를 줄이는 것이 좋을지, 같은 기능을 할지언정 따로 두는 게 더 좋을지 의문이다. 혹시 아는 분이 계시다면 댓글 부탁드려요 😄
